# Contributing to Reliability Lab Setup Guide

Thank you for your interest in improving the Reliability Lab!

## Ways to Contribute

### 1. Improve Documentation
- Fix typos or unclear explanations
- Add examples or diagrams
- Expand troubleshooting sections

### 2. Share Your Experience
- Add to FAQ based on your questions
- Contribute tips to TIPS.md
- Write about challenges you faced

### 3. Enhance Setup Guide
- Improve setup steps
- Add platform-specific instructions
- Create helper scripts

### 4. Update Cost Information
- Report current AWS pricing
- Share optimization techniques
- Document cost gotchas

## How to Contribute

### Small Changes (typos, links)
1. Click "Edit" on the file in GitHub
2. Make your changes
3. Submit pull request

### Larger Changes
1. Fork the repository
2. Create a branch: `git checkout -b improve-week-1-docs`
3. Make your changes
4. Test if applicable
5. Commit: `git commit -m "docs: improve Week 1 setup clarity"`
6. Push: `git push origin improve-week-1-docs`
7. Create Pull Request

## Commit Message Format

Use conventional commits:

- `docs:` Documentation changes
- `feat:` New features
- `fix:` Bug fixes
- `chore:` Maintenance tasks

Examples:
```
docs: add troubleshooting for EKS auth errors
feat: add cost estimation script
fix: correct kubectl command in Week 1 guide
chore: update dependencies
```

## Pull Request Process

1. **Describe your changes** clearly in PR description
2. **Link related issues** if applicable
3. **Test your changes** if they involve code/scripts
4. **Update documentation** if you change functionality
5. **Respond to feedback** from maintainers

## What to Contribute

**High Value:**
- Troubleshooting guides for common errors
- Cost optimization discoveries
- Platform-specific setup instructions (Windows, M1 Mac, etc.)
- Claude prompt examples that worked well

**Also Welcome:**
- Typo fixes
- Broken link repairs
- Formatting improvements
- Translation efforts

**Please Don't:**
- Add promotional content
- Submit AI-generated content without review
- Make large refactors without discussion first

## Code of Conduct

### Be Respectful
- Assume good intent
- Provide constructive feedback
- Welcome newcomers

### Be Helpful
- Answer questions when you can
- Share what worked for you
- Link to resources

### Be Professional
- No harassment or discrimination
- Keep discussions on-topic
- Respect maintainer decisions

## Questions?

- **Before contributing:** Check existing Issues and PRs
- **For questions:** Open a Discussion
- **For bugs:** Open an Issue
- **For improvements:** Open a PR
